## Why

当前测试脚本以扁平列表管理，所有脚本混在一起，无法按业务维度（HTTP接口测试、功能测试、性能测试等）分类组织。用户需要测试套概念来批量管理一组相关的测试脚本，并支持 ZIP 打包上传、套内统一执行。

## What Changes

### 新增功能

- **测试套管理**
  - 测试套卡片网格视图（默认入口）
  - 套名称唯一性校验
  - 套的创建（ZIP 上传）、重命名、删除（先删套内脚本再删套）
  - 套内脚本汇总信息展示（脚本数、总行数、最新上传时间）

- **套内脚本管理**
  - 点击套卡片进入套内脚本表格视图
  - 套内单独上传脚本文件（不支持批量 ZIP）
  - 套内脚本的一键执行（创建任务，执行套内所有脚本）
  - 套内脚本的执行、预览、删除

- **ZIP 批量上传**
  - 上传测试套 ZIP 文件，自动解析并导入脚本
  - ZIP 内为纯文件列表（.py/.sh/.js）
  - 套状态：成功 / 上传中（临时）/ 上传失败（套不存在）

- **任务维度的执行管理**
  - 一次"一键执行"产生一个任务记录
  - 任务包含：suite_id、触发时间、汇总统计（总数/成功/失败）
  - 执行记录关联到任务（task_id）
  - 左侧任务下拉框按套+时间组织，显示任务摘要
  - 支持筛选任务
  - 支持停止整个任务

- **导出功能**
  - 套卡片支持导出该套所有脚本为 ZIP

### 数据模型变更

- 新增 `test_suites` 表
- `test_scripts` 表增加 `suite_id`（所属套）、`uuid`（全局唯一索引）字段
- 新增 `test_tasks` 表（任务）
- `test_executions` 表增加 `task_id` 字段

### 前端页面变更

- 脚本管理：默认显示测试套网格视图
- 点击套卡片进入套内脚本表格（复用原有表格功能）
- 执行管理：左上角任务下拉框，按套+时间筛选

## Capabilities

### New Capabilities

- `test-suite`: 测试套的创建、编辑、删除、列表管理
- `suite-script`: 套内脚本的管理和执行
- `test-task`: 任务粒度的执行记录管理，支持停止任务

## Impact

- **后端**: 新增 Suite CRUD API、ZIP 上传/导出 API、Task API
- **数据库**: 新增表、字段变更，需要重建数据
- **前端**: 新增套管理视图、重构执行管理页面
- **现有数据**: 用户表示会删库重建，无需迁移
