## Why

测试脚本上传时需要用户手动填写脚本名称，但上传的脚本文件名本身已经是标准命名格式（如 TC_SBG_Func_GIDS_001_004.py），这个额外的输入框增加了操作复杂度，且容易出现名称不一致的问题。

通过自动解析上传文件的文件名作为用例名称，可以简化上传流程，减少用户操作步骤。

## What Changes

- 前端：移除脚本上传表单中的"名称"输入框
- 后端：上传接口改为从文件名自动提取用例名称，不再依赖请求参数中的 name 字段
- 文件名解析规则：从上传文件名中提取标准测试用例编号和描述作为名称

## Capabilities

### New Capabilities

- `script-auto-naming`: 上传脚本时自动从文件名解析用例名称，不再需要手动填写

## Impact

- 影响前端脚本上传组件和表单验证逻辑
- 影响后端上传 API 的参数处理
- 用户上传体验简化
