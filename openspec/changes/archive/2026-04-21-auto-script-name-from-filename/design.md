## Context

当前上传测试脚本时，前端要求用户填写脚本名称（如"IMSI非空验证测试"），后端接收后存入数据库。但实际上传的脚本文件名本身已经是标准命名（如 `TC_SBG_Func_GIDS_001_004_optimized.py`），包含了用例编号和描述信息。

这个额外的手动输入步骤增加了用户负担，且容易导致名称与实际脚本不符的问题。

## Goals / Non-Goals

**Goals:**
- 移除前端脚本上传表单中的"名称"输入框
- 后端自动从上传文件名解析用例名称
- 保持上传流程的完整功能（文件校验、存储等）

**Non-Goals:**
- 不修改现有脚本文件的存储结构
- 不改变脚本执行逻辑
- 不涉及脚本内容的解析

## Decisions

### 1. 文件名解析规则

**决策**: 从上传文件名中提取用例名称，移除文件扩展名

**解析逻辑**:
- `TC_SBG_Func_GIDS_001_004_optimized.py` → `TC_SBG_Func_GIDS_001_004_optimized`
- `TC_SBG_Func_GIDS_001_005.py` → `TC_SBG_Func_GIDS_001_005`

**备选方案**:
- 解析文件内部 `if __name__ == "__main__"` 后的 docstring → 过于复杂，依赖脚本内容格式
- 用户手动输入 → 回到原有设计

### 2. API 参数变更

**决策**: 后端上传接口忽略前端传来的 name 参数，改为自动生成

**变更**:
- 前端 `POST /api/v1/scripts/upload` 不再传递 `name` 参数
- 后端从 `file.Filename` 提取名称存入数据库

## Risks / Trade-offs

- **风险**: 如果上传的文件名不符合标准格式（如 `test.py`），名称将只是 `test`
- **缓解**: 说明文档中告知用户应使用标准命名格式上传
