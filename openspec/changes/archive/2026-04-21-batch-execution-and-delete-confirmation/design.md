## Context

当前平台功能:
- 脚本管理: 上传、执行单个脚本、删除单个脚本
- 执行管理: 查看执行记录

缺少的能力:
- 批量执行多个脚本
- 批量删除脚本
- 删除确认机制

## Goals / Non-Goals

**Goals:**
- 提供一键批量执行所有已上传脚本的功能
- 支持脚本管理页面的批量删除，需手动输入 "delete" 确认
- 执行管理页面单记录删除增加确认对话框

**Non-Goals:**
- 不支持选择性批量执行（本次仅实现全部执行）
- 不修改现有单记录执行逻辑

## Decisions

### 1. 批量执行 API 设计

**方案**: 新增 `POST /api/scripts/batch-execute` 接口

- 前端调用一次 API，后端遍历所有脚本调用现有执行逻辑
- 返回批量执行的整体结果（成功/失败数）

### 2. 批量删除确认机制

**方案**: 前端 Modal 输入框 + 后端二次校验

- 前端弹出 Modal，显示选中脚本数量
- 用户必须在输入框输入 "delete" 才能点击确认
- 后端删除前再次校验，确认操作意图

### 3. 执行记录删除确认

**方案**: 前端确认对话框（Ant Design Modal.confirm）

- 点击删除按钮触发 Ant Design Modal.confirm
- 用户点击"确定"后才执行删除 API

## Risks / Trade-offs

- [风险] 批量执行耗时长 → 前端显示加载状态，后端记录每个脚本的执行结果
- [风险] 批量删除误操作 → 要求手动输入 "delete" 提供安全保障
